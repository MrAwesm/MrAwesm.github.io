## 1. 动机：为什么需要关心频率？ {#section-1}

### 1.1 洗衣机为什么会跳舞？

洗衣机脱水时有时会剧烈抖动甚至"走路"移位。这不是坏了，是**共振**。

每台机器都有一个"讨厌的频率"——以这个频率去推它，它会越抖越厉害，就像推秋千，每次都在最高点推一把，越荡越高。

![共振现象示意图]({{ site.baseurl }}/images/posts/nudft-frequency/resonance.svg)
*图1：机械共振现象——当外部激励频率接近系统固有频率时，振动幅度急剧放大*

机器人也一样。电机、减速器、连杆组成的机械结构有自己的**固有频率**。如果关节的加速度恰好在这个频率附近来回振荡，即使振荡的幅度不大（满足加速度限幅），也可能激发结构共振，导致末端抖动、齿轮磨损、甚至损坏。

### 1.2 限幅 ≠ 限频

运动控制器通常会给加速度设一个上限：不允许超过 ±a_max。这就像限速 120km/h。

但限速不禁止你在 0 和 119 之间反复踩油门刹车——平均速度合规，乘客却被颠吐了。

![限幅与限频对比]({{ site.baseurl }}/images/posts/nudft-frequency/limit-comparison.svg)
*图2：限幅约束允许高频振荡（左），而限频约束抑制振荡（右）*

**频率约束**就是补充的一句话：加速度中高于某个频率的振荡成分，给我压下去。

## 2. 傅里叶变换的直觉：负号从哪来？ {#section-2}

要约束频率，首先要**测量**频率。这就需要傅里叶变换。

### 2.1 公式

```
频率 f 处的频谱 = ∫ g(t) · e^{-j2πft} dt
```

为什么指数上有个**负号**？这是整个频域分析的基石。

### 2.2 "迎面跑才能看清"

![傅里叶变换直觉]({{ site.baseurl }}/images/posts/nudft-frequency/fourier-intuition.svg)
*图3：傅里叶变换的核心直觉——迎面跑才能看清目标频率*

你站在路边，看一群人骑共享单车。你想知道："队伍里有几个人骑得正好是 10km/h？"

策略：**自己也骑一辆车，以 10km/h 同方向行驶**。那些速度正好是 10km/h 的人，对你来说是**静止**的——没有相对运动。其他速度的人要么追过你，要么被你甩开——对你来说是**运动**的。

"静止"= 容易识别。"运动"= 混在一起看不清。

傅里叶变换的核心策略就是如此：**把自己调到跟目标频率一样的速度、反方向旋转，让目标"停下来"看清它。**

### 2.3 数学验证

假设信号 g(t) 里藏着一个频率为 f₀ 的纯正弦成分：

```
信号成分：e^{+j2πf₀t}     ← 以 f₀ 正向旋转
检测器：  e^{-j2πf₀t}     ← 以 f₀ 反向旋转（负号！）
```

相乘：

```
e^{+j2πf₀t} · e^{-j2πf₀t} = e^{j·0} = 1    ← 旋转停止，变成常数！
```

常数的积分非零——**"检测到了"。**

频率不匹配时（f ≠ f₀）：

```
e^{+j2πf₀t} · e^{-j2πft} = e^{+j2π(f₀-f)t}    ← 还在转，速度变了
```

旋转的东西在整周期上积分**等于零**——**"检测不到"。**

### 2.4 如果没有负号？

```
e^{+j2πf₀t} · e^{+j2πf₀t} = e^{+j4πf₀t}    ← 转得更快了！
```

没有抵消，反而加速。积分还是零——**什么都检测不到**。

就像跟小偷同方向跑，他永远在你前面，看不清脸。**必须迎面跑，才能在擦肩而过的瞬间看清。**

### 2.5 "抵消"不是"消失"

日常语言里"抵消"= 消失。但这里的抵消 = **旋转停下来变成常数**。恰恰因为停下来了，积分才非零，才被"看见"。

> 傅里叶变换不是在"消灭"频率成分，而是在"冻结"它。旋转的冻结不住，积出来是零；不旋转的冻结住了，积出来就是它的强度。

## 3. 从连续到离散：DFT的前世今生 {#section-3}

### 3.1 时间离散化

计算机无法处理连续信号。在时间轴上每隔 Δt 秒取一个点：

```
t_n = n · Δt        n = 0, 1, ..., N-1

连续积分 → 离散求和：
∫ g(t) · e^{-j2πft} dt  →  Σ g_n · e^{-j2πf·t_n}
```

### 3.2 采样率与奈奎斯特极限

![采样与混叠现象]({{ site.baseurl }}/images/posts/nudft-frequency/sampling-aliasing.svg)
*图4：采样混叠现象——高速旋转的风扇在低采样率下看起来静止*

用手机拍慢动作视频：正常 30 帧/秒，慢动作 240 帧/秒。帧率 = 采样率。

假设一个风扇在转，你每隔 Δt 怼一张照片：

- 风扇 1 圈/s，你 4 张/s → 能看出在转 ✓
- 风扇 4 圈/s，你 4 张/s → 每次值到的都是同一个位置，**看起来静止** ✗

一个高速旋转的风扇，在低采样率下看起来是静止的。这就是**混叠**——高频伪装成了低频或零频，就像汽车轮子在视频里看起来倒转。

**奈奎斯特定理：采样率 Fs = 1/Δt 能分辨的最高频率是 Fs/2 = 1/(2Δt)。**

### 3.3 频率离散化

N 个采样点最多能分辨多少个频率？两个因素决定：

**频率分辨率**（相邻频率的最小间隔）由**总观察时间**决定：

```
T = N · Δt
频率分辨率 = 1/T = 1/(N·Δt)
```

直觉：你观察一个摆钟 10 秒，看到摆了 5 次，估计频率 0.5Hz。但它实际可能是 0.5001Hz——只看了 10 秒，分辨不出 0.0001Hz 的差别。要看更细，就得观察更久。

**频率范围**由采样率决定：

```
最高频率 = 1/(2Δt)
```

**频率网格**：在频率范围内，以分辨率为间距等间距取点：

```
f_m = m / (N·Δt)        m = 0, 1, ..., N-1

f_0 = 0                  ← DC 分量（不旋转）
f_1 = 1/(N·Δt)          ← 第一个频率
...
f_{N/2} = 1/(2Δt)       ← 最高频率（奈奎斯特）
f_{N/2+1}...f_{N-1}     ← 负频率（共轭对称）
```

### 3.4 Δt 的约化——标准 DFT 的诞生

将 t_n = n·Δt 和 f_m = m/(N·Δt) 代入离散傅里叶变换的核：

```
f_m · t_n = [m/(N·Δt)] · [n·Δt] = m·n / N
```

**Δt 在分子分母中约掉了！**

```
DFT(m) = Σ_{n=0}^{N-1} g_n · e^{-j2π·m·n/N}
```

这就是教科书里的标准 DFT 公式。它"看不到"时间步长——因为均匀采样时，**时间信息完全编码在采样索引 n 里了**。

### 3.5 数值验证

Δt = 0.001s，N = 8：

```
时间窗口 T = 0.008s
频率分辨率 = 125 Hz
频率网格：f_m = m × 125 Hz → [0, 125, 250, 375, 500, 625, 750, 875]

验证第 m=2, n=3 的核元素：
按定义：e^{-j2π·250·0.003} = e^{-j2π·0.75}
按化简：e^{-j2π·2·3/8}     = e^{-j2π·0.75}    ✓ 完全一致
```

## 4. 当时间刻度不再均匀：NUDFT {#section-4}

到目前为止，一切都建立在"等间距采样"上。但如果时间刻度是**不均匀**的呢？

### 4.1 一个真实场景：变速回放

![时间刻度对比]({{ site.baseurl }}/images/posts/nudft-frequency/time-scale-comparison.svg)
*图5：均匀采样（上）与非均匀采样（下）的时间刻度对比*

想象你在看一部**变速播放**的电影——有时快进，有时慢放。每一帧在"播放进度条"上是均匀的（第 1 帧、第 2 帧、第 3 帧...），但实际经过的时间忽长忽短。

你想分析画面内容的**闪烁频率**（相对于物理时间的频率），就不能假装时间轴是均匀的。

在机器人轨迹规划中也有类似情况：规划算法在"参数空间"上均匀采样，但由于机器人运动速度在变化（加速、减速），对应的**物理时间采样是不均匀的**。有些地方密（低速段，物理时间走得慢），有些地方疏（高速段，物理时间走得快）。

```
均匀采样（DFT 适用）：  |←Δt→|←Δt→|←Δt→|←Δt→|         标准尺子

非均匀采样（需要 NUDFT）：|←短→|←──长───→|←短→|←─中─→|   变形的尺子
```

### 4.2 逐步对比：DFT 的每一步在非均匀时发生了什么

**采样率**：

```
均匀时：  Fs = 1/Δt（唯一确定）
非均匀时：没有统一定义，每个间隔都不同
          有效上限 ≈ 1 / (2·最宽间隔)
```

非均匀时每个间隔不同，**最宽的间隔才是瓶颈**——那里采样最稀疏，最容易丢失信息。就像一条链子的强度取决于最弱的环节。

**频率分辨率**：

```
均匀时：  Δf = 1/(N·Δt)
非均匀时：Δf ≈ 1 / 总观察时间
```

频率分辨率仍然由**总时间跨度**决定。你观察一个摆钟 10 秒，不管每秒看一次还是时疏时密地看，频率判断精度都约 1/10 Hz。

**频率网格**：

```
均匀时：  f_m = m/(N·Δt)（自动决定，因为 Δt 约掉了）
非均匀时：f_m 可以自由选择（只需覆盖你关心的频段）
```

这是最关键的分岔点——均匀时频率网格是"自动的"，非均匀时需要你"手动选"。

### 4.3 Δt 为什么约不掉了

回到核矩阵元素 K(m,n) = e^{-j2π·f_m·t_n}。

**均匀时**：

```
t_n = n · Δt（整齐的等差数列）
f_m = m / (N·Δt)（也是等差数列，且含 Δt）

f_m · t_n = [m/(N·Δt)] · [n·Δt] = m·n/N    ← Δt 约掉了！
```

**非均匀时**：

```
t_n = 各不相同的真实时间点，没有统一的 Δt 乘在 n 上

f_m · t_n = f_m · t_n    ← 没有任何东西可以约！
```

举个例子，假设物理时间节点如下：

```
t₀ = 0
t₁ = 0.00133
t₂ = 0.00333
t₃ = 0.00433
```

这些数之间没有统一的乘法因子。你无法把它们写成"n × 某常数"的形式。所以 Δt 约不掉，**公式必须带着真实的 t_n 原样保留**。

这个保留了真实时间节点的变换，就是 **NUDFT**（Non-Uniform DFT）：

```
NUDFT(m) = Σ_{n=0}^{N-1} g_n · e^{-j2π·f_m·t_n}
```

### 4.4 完整对比

| 维度 | DFT（均匀采样） | NUDFT（非均匀采样） |
|------|-----------------|---------------------|
| 时间节点 | t_n = n·Δt（等差数列） | t_n 各不相同 |
| 采样率 | 1/Δt（唯一） | 无统一定义 |
| 频率分辨率 | 1/(N·Δt) | 1/总时间跨度 |
| 频率网格 | 自动决定（Δt 约掉） | 自由选择 |
| 核矩阵 | e^{-j2πmn/N} | e^{-j2πf_m·t_n} |
| 时间步长 | 公式中消失 | 保留 |

> **DFT 是 NUDFT 在"假设时间刻度均匀"时的特例——Δt 约掉后公式变简洁了，让人误以为 DFT 与物理时间无关。NUDFT 把被约掉的物理时间放回来，仅此而已。**

## 5. 频域代价：如何把抑制高频变成数学优化问题 {#section-5}

### 5.1 频域权矩阵

![频域代价构造流程]({{ site.baseurl }}/images/posts/nudft-frequency/frequency-cost-flow.svg)
*图6：频域代价的三步构造流程——时域→频域→加权→回时域*

我们想要一个量来衡量"信号中有多少频率落在惩罚区间内"。

用 NUDFT 核 K_nu 和一个频率滤波器 F（指定哪些频率要惩罚），可以构造一个权矩阵：

```
W = real(K_nu^H · diag(|F|²) · K_nu)
```

这个过程可以理解为三步翻译：

```
时域信号 ──[K_nu]──→ 频域 ──[diag(|F|²)]──→ 加权 ──[K_nu^H]──→ 回到时域
                        ↑                          ↑
                   "翻译成频率"              "按频率惩罚"
```

翻译回来后，我们就得到了一个时域权矩阵 W，使得：

```
信号的频率惩罚能量 = θ̈^T · W · θ̈
```

这正是论文提出的频域代价，推广到了非均匀采样的情形。

### 5.2 为什么这等于"滤波后求能量"

频域加权能量 = 时域滤波后能量。这是 **Parseval 定理**的内容。

用类比理解：你要统计"班级中身高 180-190cm 的同学的总身高"。

- **方法 A（频域）**：画身高分布直方图，在 180-190 区间乘以权重 1，其余乘以 0，求和。
- **方法 B（时域）**：挨个量身高，只记录 180-190 范围内的，加起来。

两种方法结果一样。方法 A 就是 `θ̈^T · W · θ̈`（权矩阵），方法 B 就是"先滤波再求能量"。

### 5.3 怎么塞进优化器？

优化器不直接操作加速度 θ̈，它操作的是**决策变量** Z（比如未来的控制指令序列）。但加速度是 Z 的线性函数：

```
θ̈ = C · Z + d
```

- C 是一个矩阵，把"控制指令"翻译成"加速度"
- d 是自由响应（如果什么都不做，加速度会是多少）

代入频率代价：

```
J_freq = (CZ + d)^T · W · (CZ + d)
       = Z^T · (C^TWC) · Z  +  2 · (C^TWd)^T · Z  +  常数
              ↑                      ↑
           ΔH（二次项）           Δf（一次项）
```

这就是一个关于 Z 的标准二次型！直接叠加到优化器已有的代价函数上即可：

```
H_new = H_old + ΔH
f_new = f_old + Δf
```

**不增加任何约束，不改变优化问题的结构。**

### 5.4 Δf 为什么不能省？

如果只加 ΔH 不加 Δf，会发生什么？

类比：你开车上坡，车已经在晃了。

- **只有 ΔH**：给油门加了弹簧，限制踩的力度。但车已经晃了，限制力度只能防止晃得更厉害，**不能主动减小已有的晃动**。
- **有 ΔH + Δf**：不仅限制了力度，还给出了一个"反向推力"，**主动把已有晃动按回去**。

Δf 的物理含义：自由响应 d 中已经含有高频成分（来自上一时刻的惯性），Δf 项告诉优化器："当前预测已经有高频抖动了，你需要主动产生反相的控制来抵消它。"

## 6. 低秩加速：抓住主要的振动模式 {#section-6}

### 6.1 计算瓶颈

W 是 N×N 的矩阵（N 是预测步数）。对每个关节要计算 C^T·W·C，当关节多、预测步数多时，计算量可能接近控制周期的极限。

### 6.2 关键观察：带阻滤波器的权矩阵是低秩的

带阻滤波器只惩罚一个窄频带。比如 N=80 个频率采样点中，只有 5-10 个落在惩罚区间内。

这意味着 W 的大部分特征值都接近零，只有少数几个显著。就像一个交响乐团，虽然有很多乐器，但真正在演奏主旋律的可能只有三五种。

### 6.3 低秩分解

```
W ≈ V_r · D_r · V_r^T     （只保留前 r=5~10 个特征值）

V_r：N × r 的矩阵（r 个主要"振动模式"）
D_r：r × r 的对角矩阵（每个模式的权重）
```

用这个小得多的近似替代完整的 W：

```
ΔH = Σ_j (D_r^{1/2} · V_r^T · C_j)^T · (D_r^{1/2} · V_r^T · C_j)
   = Σ_j B_j^T · B_j
```

原来要算 N×N 的矩阵乘法，现在只需 r×N 的小矩阵乘法。**加速约 10 倍。**

### 6.4 物理类比：模态分析

描述一座桥的复杂振动，不需要记录桥上每个点每时每刻的位移。结构动力学告诉我们，任何复杂振动都可以分解为少数几个**主振动模式**（模态）的叠加。抓住前几个模态就够了。

低秩分解本质上就是**对权矩阵做了一次模态分析**——只保留对频率惩罚真正有贡献的少数几个"模式"，丢弃其余。

## 7. 总结：完整推理链路 {#section-7}

![完整推理链路]({{ site.baseurl }}/images/posts/nudft-frequency/complete-chain.svg)
*图7：从傅里叶变换到实时控制的完整推理链路*

```
连续傅里叶变换：  ∫ g(t)·e^{-j2πft} dt       ← "迎面跑才能看清"
        ↓ 时间离散化（均匀 Δt）
标准 DFT：        Σ g_n·e^{-j2πmn/N}          ← Δt 约掉了，与物理时间脱钩
        ↓ 时间非均匀（t_n 各不相同）
NUDFT：           Σ g_n·e^{-j2πf_m·t_n}       ← 物理时间放回来了
        ↓ 构造权矩阵
频域权矩阵：      W = real(K_nu^H·diag(F²)·K_nu)  ← "频率筛子"
        ↓ 建立加速度→决策变量的线性映射
二次代价：        Z^T·ΔH·Z + Δf^T·Z           ← 叠加到优化器，不改结构
        ↓ 低秩分解（模态分析）
加速：            N×N → N×r（r≈8）             ← 10 倍加速，满足实时性
```

整个方案的精髓：**不改盘子，只加调料。** 优化器的盘子和做法都不变，只是在代价函数里多加了一个"别振荡太快"的偏好。

> **核心结论**：NUDFT 将被均匀采样"约掉"的物理时间重新引入频域分析，使我们能够在非均匀采样的轨迹规划中精确构造频率约束代价，并通过低秩分解实现实时计算。
